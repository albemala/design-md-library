# Design.md Library

A public library of AI-ready `DESIGN.md` files, reusable design systems for generating consistent UI with AI.

## What is this repository?

This repository is an open collection of high-quality `DESIGN.md` files.

Each file defines a complete design system in a format that AI tools can understand and apply when generating user interfaces. The goal is to make AI-generated UI more consistent, usable, and production-ready.

This library is:

* Open and community-driven
* Focused on quality over quantity
* Designed for immediate use (copy, download, plug into your workflow)

## What are `DESIGN.md` files?

`DESIGN.md` is an emerging open standard for defining design systems in a way that both humans and AI can understand.

Each file combines:

* **Structured design tokens** (colors, spacing, typography, etc.)
* **Design rules and constraints** (guidelines, do’s and don’ts)

This allows AI tools to generate UI that follows a coherent system instead of making arbitrary design decisions.

**Learn more:**

* [What is DESIGN.md?](https://stitch.withgoogle.com/docs/design-md/overview/)
* [DESIGN.md Specification](https://github.com/google-labs-code/design.md)

## How to use `DESIGN.md` files

You can use these files with AI tools that support contextual design input, such as coding assistants and generative UI systems.

Typical workflow:

1. Pick a `DESIGN.md` file from this library
2. Copy or download it
3. Provide it as context to your AI tool (for example alongside your prompt or in a rules or config file)
4. Generate UI that follows the defined design system

The result is more consistent layouts, better visual hierarchy, and fewer design inconsistencies.
