Design Systems

The quiet craft of editorial design systems

MK
Maria Keppler 12 min read May 8, 2026 New

Content is sovereign. The interface exists to present words and images at their best and then disappear. This principle guides every decision in the Media & Editorial design system — a system built for blogs, magazines, and newsletters where sustained reading is the primary job-to-be-done.

The tone is readable, elegant, editorial, and unhurried. Navigation and platform voice stay quiet so the writer's voice leads. Every component decision starts with a single question: does this make reading better?

Typography as architecture

Typography is optimized for long-form reading, not density. Serif is mandatory for article content and major headlines — Lora for display and headings, Source Serif 4 for body text. Sans-serif is reserved for navigation, controls, labels, and metadata through Inter.

Body copy is set to 19px with 1.8 line height and a fixed reading measure of 680px. Text is always left-aligned; justified body text is disallowed. These constraints aren't limitations — they're the guardrails that keep reading comfortable across every article.

The restrained palette

The palette is intentionally restrained: near-black text on white reading surfaces with one editorial accent — accent red. Reading surfaces stay white in default mode to maximize readability. The accent is used sparingly: active nav, pull quotes, hover states, selected highlights.

Metadata and supporting text move down a neutral scale without reducing contrast below accessible thresholds. Three reading modes — light, sepia, and dark — shift only background, text, and accent relationships, never layout or typographic structure.

Code in context

Technical articles need clear code presentation. Code blocks use small rounding for scannability while keeping the editorial feel:

// Design token configuration
const tokens = {
  bodyMaxWidth: '680px',
  imageMaxWidth: '960px',
  bodyLineHeight: 1.8,
  bodyFontSize: '19px',
  accentColor: '#D64045'
};

Inline code like lineHeight: 1.8 uses a subtle background tint to distinguish from body text without breaking the reading rhythm. This is how technical content integrates into an editorial flow without visual disruption.

Layout that breathes

Layout enforces editorial rhythm through narrow reading measure, image breakout, and generous vertical spacing. The article text column maxes at 680px. Images can break out to 960px — creating pacing contrast that prevents visual monotony during long reads. Overall page content caps at 1200px.

Base spacing uses the editorial scale — 16px to 96px — with larger vertical gaps than typical app UIs. No sidebar is shown while reading article content. The page respects three breakpoints: mobile (<640px) with full-width images, tablet (640–1024px) with two-column grids, and desktop (>1024px) with three-column layouts.
